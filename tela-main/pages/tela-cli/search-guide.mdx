# TELA-CLI Search Guide

Master the powerful search capabilities of TELA-CLI to discover applications, analyze contracts, and navigate the TELA ecosystem.

<Callout type="info" emoji="üîç">
**Prerequisites:** Gnomon indexer must be running. See [Gnomon Guide](/tela-cli/gnomon-guide) for setup.
</Callout>

## Search Command Overview

TELA-CLI provides 15+ search variants for discovering content:

| Search Type | Command | Purpose |
|-------------|---------|---------|
| **All Content** | `search all` | Browse everything |
| **By SCID** | `search scid [scid]` | Specific contract |
| **By Type** | `search docs [type]` | Filter by file type |
| **Applications** | `search indexes` | Find TELA apps |
| **Libraries** | `search libs` | Find reusable code |
| **By Name** | `search durl [name]` | Search by dURL |
| **By Variable** | `search key [key]` | Find contracts with key |
| **By Value** | `search value [val]` | Find contracts with value |
| **By Author** | `search author [addr]` | Developer's content |
| **Code Analysis** | `search code [scid]` | View contract code |
| **Code Search** | `search line [text]` | Find code patterns |
| **Ratings** | `search ratings [scid]` | View all ratings |
| **Your Content** | `search my docs/indexes` | Your deployments |

## Quality Filtering

### Min-Likes System

Filter search results by rating percentage:

```bash
# Exceptional content only (90%+)
¬ª search min-likes 90
¬ª search indexes

# High quality (70%+)
¬ª search min-likes 70
¬ª search all

# Average and above (60%+)
¬ª search min-likes 60
¬ª search docs

# All content (no filter)
¬ª search min-likes 0
¬ª search all
```

**Rating Scale:**
- **90-99%** - Exceptional, best-in-class
- **70-89%** - Good to very good
- **60-69%** - Average quality
- **50-59%** - Needs improvement
- **0-49%** - Issues present

### Progressive Discovery

```bash
# Start with exceptional
¬ª search min-likes 90
¬ª search indexes
# Found 5 exceptional apps

# Expand to very good
¬ª search min-likes 75
¬ª search indexes
# Found 23 apps

# Include good content
¬ª search min-likes 65
¬ª search indexes
# Found 87 apps
```

## Advanced Filtering

### Exclusion System

Blacklist specific content from search results:

**View Current Exclusions:**
```bash
¬ª search exclude view
```

**Add Exclusions:**
```bash
# Exclude test content
¬ª search exclude add test
¬ª search exclude add demo
¬ª search exclude add example
¬ª search exclude add beta

# Now search results exclude these
¬ª search all
# No results with "test", "demo", "example", or "beta" in dURL
```

**Remove Exclusions:**
```bash
# Remove specific exclusion
¬ª search exclude remove test

# Remove all exclusions
¬ª search exclude clear
```

**Use Cases:**
- Filter out test deployments
- Hide work-in-progress apps
- Focus on production content
- Reduce noise in search results

### Combined Filtering

```bash
# High-quality, production content only
¬ª search min-likes 75
¬ª search exclude add test
¬ª search exclude add demo
¬ª search indexes

# Results: Only 75%+ rated, production apps
```

## Search Result Interpretation

### Understanding Output

**INDEX Search Result:**
```
‚≠ê 87% | myapp.tela
Type: INDEX | Author: deto1qy...abc | Height: 12,345
15 DOCs | MODs: vsoo,txdwd | Commits: 3
```

**Fields Explained:**
- `‚≠ê 87%` - Average rating (87% positive)
- `myapp.tela` - dURL identifier
- `Type: INDEX` - Contract type
- `Author: deto1qy...` - Deployer's address (truncated)
- `Height: 12,345` - Deployment block height
- `15 DOCs` - Number of files in app
- `MODs: vsoo,txdwd` - Enabled TELA-MOD-1 modules
- `Commits: 3` - Number of updates made

---

**DOC Search Result:**
```
‚≠ê 92% | index.html (homepage.tela)
Type: TELA-HTML-1 | Author: deto1qy...def | Height: 11,000
Size: 15.2 KB | SubDir: / | Compressed: Yes
```

**Fields Explained:**
- `index.html` - File name
- `homepage.tela` - dURL
- `TELA-HTML-1` - File type
- `Size: 15.2 KB` - File size
- `SubDir: /` - Directory location
- `Compressed: Yes` - Gzip compression used

## Search Troubleshooting

### No Results Found

**Diagnostic:**
```bash
# 1. Check Gnomon is running
¬ª info
# Look for [G:‚ñ≤]

# 2. Verify sync completed
¬ª search all
# Should show some results if synced
```

**Solutions:**
```bash
# 1. Start Gnomon if offline
¬ª gnomon start

# 2. Wait for sync
# Be patient, especially on mainnet

# 3. Check network
¬ª endpoint mainnet
¬ª gnomon start

# 4. Try adding specific known SCID
¬ª gnomon add [known-scid]
¬ª search scid [known-scid]
```

---

### Unexpected Results

**Issue:** Search returns content you didn't expect

**Checks:**
```bash
# 1. View current filters
¬ª search min-likes 0  # Check current threshold
¬ª search exclude view  # Check exclusions

# 2. Reset filters
¬ª search min-likes 0   # No rating filter
¬ª search exclude clear # Clear all exclusions

# 3. Search again
¬ª search all
```

---

### Slow Search Performance

**Solutions:**
```bash
# 1. Reduce page size
¬ª page-size 10

# 2. Use specific searches instead of 'all'
¬ª search indexes  # Instead of search all
¬ª search docs HTML  # Instead of search docs

# 3. Apply filters
¬ª search min-likes 70
# Fewer results = faster display
```

## Best Practices

<Callout type="tip" emoji="üí°">
**Search Best Practices**

1. **Start broad, narrow down** - Begin with `search all`, then apply filters
2. **Use quality filters** - `min-likes 70` for good content
3. **Exclude test content** - Add exclusions for cleaner results
4. **Verify before use** - Check ratings and variables
5. **Clone before integrating** - Test libraries locally first
6. **Respect ratings** - Community ratings indicate quality
7. **Search your own content** - Verify deployments indexed
</Callout>
