# TELA Platform Overview

**TELA** is the Decentralized Web Standard for DERO blockchain. Build privacy-first web applications that run entirely on-chain with zero servers, zero censorship, and complete user control.

<Callout type="success" emoji="üöÄ">
**The Future of Web3:** TELA is the ONLY platform enabling fully private, censorship-resistant web applications with encrypted storage and local execution.
</Callout>

## The TELA Difference

**Traditional Web Architecture:**
```
User ‚Üí Company Server ‚Üí Database
       ‚Üì
    Logs, Tracks, Controls Access
    Can be hacked, censored, taken offline
```

**TELA Architecture:**
```
User ‚Üí Engram (Local) ‚Üí DERO Blockchain
       ‚Üì
    Runs on your computer
    Cannot be hacked, censored, or taken offline
```

## How TELA Works: Dual-Contract System

**TELA uses two smart contract types:**

**Deployment Flow:**
```
Developer
  ‚Üì
Deploy: index.html ‚Üí DOC Contract (SCID: abc...)
Deploy: app.js    ‚Üí DOC Contract (SCID: def...)
Deploy: style.css ‚Üí DOC Contract (SCID: ghi...)
  ‚Üì
Create: INDEX Contract ‚Üí References all DOC SCIDs
  ‚Üì
DERO Blockchain (Permanent Storage)
```

**User Access Flow:**
```
User enters INDEX SCID into Engram
  ‚Üì
Engram fetches INDEX from blockchain
  ‚Üì
INDEX points to all DOC contracts (abc, def, ghi...)
  ‚Üì
Engram fetches all DOCs from blockchain
  ‚Üì
Verifies cryptographic hashes
  ‚Üì
Executes app locally in Engram
```

### üìã **INDEX Contract** (Application Manifest)

| Contains | Purpose |
|----------|---------|
| **App Name** | "My TELA App" |
| **Description** | What the app does |
| **Icon** | App thumbnail |
| **DOC References** | SCIDs of all files |
| **File Paths** | How files are organized |
| **Version** | App version number |

**Think:** Package.json for on-chain apps

---

### üì¶ **DOC Contract** (Application Files)

**Supported File Types:**

| Type | File Extension | Max Size | Use Case |
|------|----------------|----------|----------|
| **HTML** | `.html` | 18KB | Page structure |
| **CSS** | `.css` | 18KB | Styling |
| **JavaScript** | `.js` | 18KB | Logic & interactivity |
| **JSON** | `.json` | 18KB | Data & config |
| **Markdown** | `.md` | 18KB | Documentation |
| **Static Assets** | Images, fonts | 18KB | Assets |

**Size limit:** 18KB per DOC contract (blockchain constraint)  
**Solution:** Split large files across multiple DOC contracts

## Development Workflow

**From idea to deployed app:**

| Step | Action | Tool | Output |
|------|--------|------|--------|
| **A** | Write your app | Any code editor | HTML, CSS, JS files |
| **B** | Test locally | TELA-CLI serve | Preview in browser |
| **C** | Deploy files | TELA-CLI install-doc | DOC contracts (SCIDs) |
| **D** | Create manifest | TELA-CLI install-index | INDEX contract (SCID) |
| **E** | Share & access | Give users INDEX SCID | App live on blockchain! |

## Complete Architecture

**Full TELA Stack (4 Layers):**

| Layer | Component | Function |
|-------|-----------|----------|
| **üë§ User Layer** | User in Engram | Browses and interacts with apps |
| **üì± TELA Protocol** | INDEX + DOC contracts | App manifest + code files |
| **üîê Security Layer** | XSWD (port 44326) + Verification | Wallet connection + hash checking |
| **üíæ Storage Layer** | DERO Blockchain + Wallet | Immutable storage + private keys |

**Data Flow:**
```
User enters SCID
  ‚Üì
Engram fetches INDEX from blockchain
  ‚Üì
INDEX points to DOC contracts (files)
  ‚Üì
Engram fetches all DOCs from blockchain
  ‚Üì
Cryptographic verification (hash matching)
  ‚Üì
Execute locally in Engram (sandboxed)
  ‚Üì
User interacts (XSWD handles wallet operations)
```

**Every layer is decentralized, private, and verifiable!**

## Technical Foundation

### Smart Contract Architecture

| Contract Type | Mutability | Purpose | Size Limit |
|---------------|------------|---------|------------|
| **DOC Contract** | üîí Immutable | Store code files | 18KB per file |
| **INDEX Contract** | ‚úèÔ∏è Updateable | App metadata & references | Variable |

**Why two types?**
- **DOC immutability** ‚Üí Code integrity guaranteed
- **INDEX updateability** ‚Üí Apps can evolve (point to new DOCs)
- **User control** ‚Üí Can verify exact code version running

**Version Control:**
```
Version A ‚Üí DOC Contract (SCID: abc123...)
Version B ‚Üí DOC Contract (SCID: def456...)
            ‚Üë
INDEX Contract can switch between versions
All versions remain on blockchain forever
```
